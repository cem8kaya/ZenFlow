## Processing Guide

### Using Audacity (Free)

1. **Import Sound File**
   - File â†’ Import â†’ Audio
   - Load your downloaded sound file

2. **Normalize Audio**
   - Effect â†’ Volume and Compression â†’ Normalize
   - Set to -6.0 dB
   - Apply

3. **Create Seamless Loop**
   - Listen to start and end points
   - Find matching amplitude points
   - Use crossfade for smooth transition:
     - Select 3-second overlap at beginning and end
     - Effect â†’ Crossfade Tracks

4. **Test Loop**
   - Transport â†’ Loop Play
   - Listen for any clicks or pops
   - Adjust loop points if needed

5. **Export**
   - File â†’ Export â†’ Export as WAV (or MP3)
   - WAV: PCM signed 16-bit
   - MP3: 128 kbps constant bit rate

6. **Verify File Size**
   - Check file is under 10 MB
   - If too large, export as MP3 instead

### CC BY Sounds (Attribution Required)
If using CC BY licensed sounds, add attribution in app:

**Location:** Settings â†’ About â†’ Sound Credits

**Format:**
```
[Sound Name] by [Author Name]
Source: [Platform] ([License])
```

**Example:**
```
Thunderstorm Ambience by John Smith
Source: Freesound.org (CC BY 4.0)
```

### Implementation

Create `SoundCreditsView.swift` if using CC BY sounds:

```swift
struct SoundCreditsView: View {
    let attributions = [
        "Thunderstorm - John Smith (Freesound.org, CC BY 4.0)",
        "River Stream - Jane Doe (Pixabay License)",
        // Add more as needed
    ]

    var body: some View {
        List(attributions, id: \.self) { credit in
            Text(credit)
                .font(.caption)
        }
        .navigationTitle("Sound Credits")
    }
}
```

## App Size Management

### Current Status
- **3 existing sounds:** ~30 MB
- **Target for 12 new sounds:** ~120 MB
- **Total target:** < 150 MB

### Optimization Strategies

1. **Use MP3 for larger files**
   - MP3 at 128kbps is acceptable for ambient sounds
   - Can reduce file size by 80-90%

2. **Shorter loops**
   - 2-3 minute loops instead of 5 minutes
   - Ensure seamless looping to avoid repetition

3. **Lazy loading**
   - Sounds are already loaded on-demand via NSDataAsset
   - No changes needed to current implementation

4. **Asset compression**
   - Let Xcode compress assets during build
   - Enable "Compress PNG Files" in build settings (already enabled for other assets)
